## 📋 项目结构总览

```
云集智能音乐创意台/
├── dev/                        # ⭐ Git管理 - 日常开发 + 纯净整合包
│   ├── 云集智能音乐创意台-vYYYY.MM.DD.HHMM.exe  # 多个EXE版本
│   └── app/                 # 纯净整合包，完整项目文件
│       ├── main.py             # 启动器主程序
│       ├── version_manager.py  # 版本管理器
│       ├── icon.ico
│       ├── requirements.txt
│       ├── ace-step-ui/        # 青龙训练器前端
│       ├── acestep/            # 核心后端
│       └── scripts/            # 只放启动脚本（无重复项目文件！）
│
├── build/                      # ⭐ Git不管理 - 版本历史
│   └── 云集智能音乐创意台-vYYYY.MM.DD.HHMM/
│
├── pack/                       # ⭐ Git不管理 - 本地发布包
│
├── docs/                       # ⭐ Git管理 - 项目文档
│   └── 开发指南.md            # ⭐ 详细开发流程文档（必读！）
│
└── .claude/skills/             # ⭐ AI技能配置（你正在看的文件）
```

---

## 🎯 核心原则

1. **日常开发只在 dev/app/ 目录进行** - 不要直接修改其他目录
2. **dev/ 目录是 Git 管理的纯净整合包** - 推送到远程仓库
3. **build/ 目录保存版本历史** - 用于回滚和版本对比
4. **pack/ 目录存放发布给用户的纯净整合包**

---

## 📝 标准开发工作流

### 第一步：日常开发
```
在 dev/app/ 目录修改和测试代码
```

### 第二步：保存版本（测试通过后）
```
1. 在 build/ 目录创建新的时间版本文件夹
2. 把 dev/app/ 的完整内容复制到新的版本文件夹
```

### 第三步：回滚版本（如果需要）
```
从 build/ 的旧版本文件夹复制内容到 dev/app/
```

### 第四步：发布稳定版
```
从 build/ 的某个版本创建纯净整合包到 pack/
```

### 第五步：Git 提交
```
提交 dev/ 目录的更改并推送到远程仓库
```

## ⚠️ 重要提醒

1. **不要直接修改 build/ 或 pack/ 目录** - 它们只用于版本历史和发布
2. **dev/app/scripts/ 只放启动脚本** - 不要重复放置 ace-step-ui/、acestep/ 等项目文件
3. **main.py、version_manager.py、icon.ico、requirements.txt 保持在 dev/app/ 根目录** - 不要移到二级目录
4. **每次开发前，请先阅读 docs/开发指南.md**
