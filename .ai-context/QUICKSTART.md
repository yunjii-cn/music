## 💡 第四步：常见任务快速参考（1分钟）

### 场景1：修改启动器功能

1. 打开 `launcher/main.py`
2. 找到相关功能模块
3. 使用信号槽机制进行UI更新
4. 测试功能
5. 运行 `python build-correct.py` 打包

### 场景2：修复启动脚本问题

1. 确定是哪个启动脚本（2、3、4开头的.ps1文件）
2. 查看 `FILE_INDEX.md` 了解该脚本的用途
3. 修改对应的脚本
4. 测试脚本是否正常工作

### 场景3：打包和发布

1. 确保在main分支
2. 运行 `python build-correct.py`
3. 脚本会自动：
   - 构建EXE
   - 检测Git更改
   - 生成提交信息
   - 自动提交和推送

### 场景4：整理遗留文件

1. 不要直接删除！
2. 查看 `BAK/README.md` 了解归档原则
3. 根据文件类型移动到 `BAK/` 对应的子文件夹
4. 确保项目根目录保持整洁

## 🎯 分支管理速查

| 分支 | 用途 | 操作方式 |
|------|------|----------|
| **main** | 开发分支 | 自动打包上传，日常开发在此进行 |
| **beta** | 测试分支 | 手动合并和推送，用于测试验证 |
| **stable** | 稳定版分支 | 手动合并和打标签，用于正式发布 |

**工作流**: main → beta → stable

## 📁 BAK文件夹使用指南

**重要原则**: 不直接删除，而是归档

### BAK子文件夹说明

| 子文件夹 | 存放内容 |
|---------|---------|
| `packaging/` | 废弃的打包脚本 |
| `launchers/` | 废弃的启动脚本 |
| `temp_scripts/` | 废弃的测试和临时脚本 |
| `temp_python/` | 废弃的临时Python文件 |
| `temp_docs/` | 废弃的文档和临时文件 |
| `launcher_backups/` | launcher目录中的备份文件 |
| `scripts_temp/` | scripts目录中的废弃文件 |
| `utility_scripts/` | 整理工具脚本 |
| `misc_files/` | 其他零散文件 |
| `extra_configs/` | 额外配置文件 |
| `extra_docs/` | 额外文档 |
| `old_exes/` | 旧的EXE文件 |
| `linux_scripts/` | Linux脚本 |
| `extra_folders/` | 额外文件夹 |
