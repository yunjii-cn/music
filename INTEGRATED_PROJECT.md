# 整合项目说明

## 项目概述

本项目已整合官方演练场（Gradio UI）和青龙训练器（React UI）为一个统一的项目，共享资源和配置。

## 目录结构

```
e:\AI应用\qinglong-music-trainer-2.8.3/
├── acestep/          # 后端核心代码
├── ace-step-ui/      # 青龙训练器前端
├── checkpoints/      # 共享模型存储
├── config/           # 共享配置
├── data/             # 共享数据
│   ├── datasets/     # 数据集
│   ├── cache/        # 缓存
│   └── temp/         # 临时文件
├── output/           # 共享输出
├── shared/           # 共享工具
│   └── config_manager.py  # 配置管理模块
├── scripts/          # 脚本
│   └── start-all.ps1 # 统一启动脚本
└── README.md         # 项目说明
```

## 主要功能

### 1. 共享模型存储
- 所有模型存储在根目录的 `checkpoints` 文件夹中
- 两个界面共享同一套模型
- 模型只需要下载一次

### 2. 共享输出文件夹
- 所有生成的音频文件存储在根目录的 `output` 文件夹中
- 两个界面可以访问相同的输出文件

### 3. 统一配置管理
- 配置存储在 `config/config.json` 文件中
- 支持环境变量覆盖配置
- 提供统一的路径管理

### 4. 统一启动脚本
- `scripts/start-all.ps1`：同时启动所有服务
- 支持一键启动和停止

## 如何使用

### 启动所有服务

```powershell
# 在项目根目录运行
.criptstart-all.ps1
```

### 单独启动服务

#### 官方演练场（Gradio UI）
```powershell
# 在项目根目录运行
.、run_gradio.ps1
```

#### 青龙训练器（React UI）
```powershell
# 在项目根目录运行
.、run_npmgui.ps1
```

#### API 服务器
```powershell
# 在项目根目录运行
.、run_server.ps1
```

## 服务地址

- **官方演练场**：http://localhost:7860
- **青龙训练器**：http://localhost:3000
- **API 服务器**：http://localhost:8001

## 配置管理

### 配置文件
配置文件位于 `config/config.json`，可以手动编辑或通过代码修改。

### 环境变量
可以通过设置环境变量来覆盖配置：
- 环境变量名：配置键的大写形式，点号替换为下划线
- 例如：`CHECKPOINTS_DIR` 对应配置中的 `checkpoints_dir`

## 数据管理

- **模型**：`checkpoints/`
- **输出**：`output/`
- **数据集**：`data/datasets/`
- **缓存**：`data/cache/`
- **临时文件**：`data/temp/`

## 技术优势

1. **资源共享**：模型、配置、数据等资源在两个项目间共享
2. **统一管理**：通过统一的脚本和配置进行管理
3. **减少重复**：消除代码和资源的重复
4. **简化部署**：统一的启动和停止流程
5. **一致性**：两个界面使用相同的后端和资源
6. **节省存储空间**：模型只需要下载一次
7. **提高开发效率**：共享代码和配置
8. **简化维护**：统一的代码库和配置管理

## 注意事项

1. **首次运行**：第一次运行时会自动下载模型到 `checkpoints` 目录
2. **权限**：确保对所有共享目录有读写权限
3. **端口冲突**：如果端口被占用，请修改启动脚本中的端口配置
4. **依赖**：确保已安装所有必要的依赖

## 故障排除

### 模型下载失败
- 检查网络连接
- 检查磁盘空间
- 检查 `checkpoints` 目录权限

### 服务启动失败
- 检查端口是否被占用
- 检查依赖是否安装
- 查看命令行输出的错误信息

### 界面无法访问
- 检查服务是否正在运行
- 检查防火墙设置
- 检查端口配置
