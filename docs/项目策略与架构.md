## 🎯 核心策略

### 1. 单文件分发策略
- **只提供一个EXE文件**给用户
- EXE内置所有必要的核心文件
- 简洁、干净、用户体验好

### 2. 首次运行初始化策略
```
用户首次打开EXE：
├─ 自动解压必要文件到当前目录
├─ 检测硬件配置（CUDA、GPU）
├─ 检测网络环境（选择合适的源）
├─ 生成配置文件（保存用户配置）
├─ 匹配合适的环境整合包
└─ 执行环境部署
```

### 3. 智能检测策略（平衡体验与可靠性）

| 检测项 | 频率 | 说明 |
|--------|------|------|
| **硬件配置** | 首次检测 + 缓存 | 硬件很少变化，没必要每次检测 |
| **网络环境** | 需要时检测 | 只在安装/更新依赖时检测 |
| **环境完整性** | 每次快速检查（1-2秒） | 检查关键文件是否存在 |
| **版本更新** | 可选后台检查 | 用户可选择是否检查更新 |

### 4. 环境整合包策略
- **当前**：默认环境整合包（cp312+cu128+torch2.9.0）
- **未来**：根据硬件自动匹配多个不同的环境整合包
- **扩展**：支持CPU、不同CUDA版本等多种配置

## 🔧 核心技术选型

### 1. 环境管理：UV优先
| 特性 | 说明 |
|------|------|
| **UV安装位置** | 项目目录 `tools/uv/`（不污染用户目录） |
| **虚拟环境位置** | 项目目录 `.venv/`（随项目移动） |
| **Python管理** | UV自动下载、缓存、管理Python 3.12 |
| **国内源** | 清华PyPI镜像：`https://pypi.tuna.tsinghua.edu.cn/simple` |

### 2. 路径策略：相对路径
- 所有路径相对于 `self.base_dir`（EXE所在目录）
- 用户可以把项目放在任意位置
- 完全便携，随拷随用

### 3. 配置文件：JSON格式
```json
{
  "last_check": "2026-03-30T18:30:00",
  "hardware": {
    "cuda_version": "cu128",
    "gpu_model": "NVIDIA RTX 4090",
    "has_gpu": true
  },
  "network": {
    "preferred_source": "tsinghua",
    "last_speed_test": "2026-03-30T18:30:00"
  },
  "environment": {
    "version": "v2026.03.30.1820",
    "is_valid": true
  }
}
```

## 🚀 运行流程

### 流程图
```
用户打开EXE
    │
    ├─ 快速检查（1-2秒）
    │   ├─ 检查 .venv/Scripts/python.exe 是否存在？
    │   ├─ 检查 tools/uv/uv.exe 是否存在？
    │   └─ 简单测试Python是否能运行？
    │
    ├─ 判断：环境是否有效？
    │   ├─ 有效 → 直接启动主程序
    │   │
    │   └─ 无效 → 提示用户
    │           ├─ 用户选择"重新检测" → 进入首次运行流程
    │           └─ 用户选择"退出" → 退出程序
    │
    └─ 首次运行流程（环境不存在）
        ├─ 解压必要文件
        ├─ 硬件检测（CUDA版本、GPU型号）
        ├─ 网络检测（选择合适的PyPI源）
        ├─ 生成 config.json
        ├─ 匹配环境整合包
        └─ 环境部署（安装UV、创建虚拟环境、安装依赖）
```

## 💡 关键设计原则

1. **用户优先**：体验第一，简洁至上
2. **相对路径**：所有路径相对EXE所在目录
3. **智能检测**：平衡可靠性和用户体验
4. **可扩展**：为未来的环境整合包预留接口
5. **易测试**：便于模拟用户环境进行测试
