# 混合式版本管理器使用指南

## 什么是混合式版本管理器？

混合式版本管理器是一种**双模式**的版本选择界面，结合了：
- **下拉菜单** - 快速访问最近版本（3-5 个）
- **弹窗对话框** - 完整的版本管理功能

这种设计让你既能**快速切换**到最近的版本，又能**浏览完整历史**。

## 快速开始

### 1. 打开版本管理器

在 launcher 主窗口顶部，找到蓝色的 **"📦 版本管理"** 按钮，点击它。

### 2. 使用下拉菜单（快速模式）

点击按钮后，会显示一个下拉菜单，包含：


**快速操作：**

## 功能详解

### 快速切换（下拉菜单）

**适用场景：**
- 切换到最近使用的版本
- 快速回退到上一个版本
- 测试最新的 commit

**操作步骤：**
1. 点击 "📦 版本管理" 按钮
2. 在下拉菜单中选择目标版本
3. 在确认对话框中点击 "是"
4. 等待切换完成
5. 重新启动程序

**优点：**
- ⚡ 快速 - 2 次点击即可完成
- 🎯 直观 - 直接看到最近的更改
- 💡 便捷 - 无需打开额外窗口

### 完整管理（弹窗对话框）

**适用场景：**
- 浏览完整版本历史
- 查看详细的 commit 说明
- 切换到较旧的版本
- 对比不同版本

**功能按钮说明：**

| 按钮 | 功能 | 说明 |
|------|------|------|
| 🔄 刷新 | 重新加载版本列表 | 当 Git 仓库有更新时使用 |
| 👁️ 预览 | 查看版本详情 | 显示完整的 commit message |
| 🔄 切换到此版本 | 切换到目标版本 | 需要确认后会执行 |
| 📝 查看完整日志 | 查看最近 50 条记录 | 纯文本格式的 git log |
| 关闭 | 关闭弹窗 | 返回主程序 |

**操作步骤：**
1. 点击 "📦 版本管理" → "查看更多版本..."
2. 在弹窗中浏览版本列表
3. （可选）点击 "👁️ 预览" 查看详情
4. 点击 "🔄 切换到此版本"
5. 在确认对话框中点击 "是"
6. 等待切换完成并重启程序

## 常见使用场景

### 场景 1: 回退到稳定版本

**问题：** 最新版本有 bug，想回到之前的稳定版本

**步骤：**
1. 点击 "📦 版本管理"
2. 如果目标版本在下拉菜单中，直接点击
3. 否则点击 "查看更多版本..."
4. 在完整弹窗中找到目标版本
5. 点击 "🔄 切换到此版本"
6. 确认后等待重启

### 场景 2: 测试最新功能

**问题：** 想尝试开发者刚刚提交的新功能

**步骤：**
1. 点击 "📦 版本管理"
2. 下拉菜单中的第一个就是最新版本
3. 点击该版本
4. 确认后重启即可体验

### 场景 3: 对比多个版本

**问题：** 想了解不同版本之间的差异

**步骤：**
1. 点击 "📦 版本管理" → "查看更多版本..."
2. 对感兴趣的版本点击 "👁️ 预览"
3. 查看完整的 commit message
4. 可以打开多个预览窗口进行对比

### 场景 4: 查找特定修复

**问题：** 记得某个修复但不确定在哪个版本

**步骤：**
1. 点击 "📦 版本管理" → "查看更多版本..."
2. 浏览版本列表查找相关 commit
3. 或使用 "📝 查看完整日志" 查看全部历史
4. 找到后切换过去验证

## 版本切换时会发生什么？

当你切换到另一个版本时，系统会自动：

### 1. 备份用户数据
```
备份位置：`.git_backup/`
包含内容：
- config/     (配置文件)
- models/     (下载的模型)
- output/     (生成的音频文件)
- .env        (环境变量)
```

### 2. 执行 Git 切换
```bash
git checkout <commit-hash>
```

### 3. 恢复用户数据
```
将备份的用户数据恢复到原位置
确保你的配置和模型不会丢失
```

### 4. 提示重启
```
切换完成后会弹出提示
点击 "确定" 后程序退出
手动重新启动即可使用新版本
```

## 注意事项

### ⚠️ 切换前请保存工作

版本切换会重启程序，请确保：
- 已保存所有配置
- 已完成当前的生成任务
- 已保存重要的日志或输出

### ⚠️ 未提交的更改

如果工作区有未提交的更改：
- Git checkout 可能会失败
- 系统会显示错误信息
- 需要先提交或暂存更改

**解决方法：**
```bash
# 查看未提交的更改
git status

# 提交更改
git add .
git commit -m "保存当前工作"

# 或者暂存更改
git stash
```

### ⚠️ 大文件处理

切换版本时如果需要下载/删除大文件：
- 可能需要较长时间
- 请确保有足够的磁盘空间
- 不要在切换过程中强制关闭程序

## 故障排除

### 问题 1: 点击按钮无反应

**检查项：**
- Git 是否已安装
- 项目是否是 Git 仓库

**解决方法：**
```bash
# 检查 Git
where git

# 检查 Git 仓库
git status
```

### 问题 2: 版本列表为空

**可能原因：**
- `.git` 文件夹损坏
- Git 命令执行失败

**解决方法：**
```bash
# 手动测试 Git 命令
git log -n 5 --pretty=format:"%h|%s|%ai"
```

### 问题 3: 切换失败

**错误信息示例：**
```
Git checkout 失败:
error: Your local changes to the following files would be overwritten by checkout
```

**解决方法：**
1. 根据错误信息提交或暂存更改
2. 重新尝试切换

### 问题 4: 切换后程序异常

**可能原因：**
- 版本之间存在不兼容的更改
- 依赖项发生变化

**解决方法：**
```bash
# 重新安装依赖
.venv\Scripts\activate
pip install -r requirements.txt

# 或者切换回之前的版本
```

## 高级技巧

### 技巧 1: 使用键盘快捷键

虽然版本管理器主要是鼠标操作，但你可以：
- `Enter` - 确认切换
- `Esc` - 取消操作
- 方向键 - 在菜单中导航

### 技巧 2: 批量测试多个版本

需要测试多个版本？

1. 打开完整弹窗
2. 对每个版本点击 "👁️ 预览" 而不是 "切换"
3. 记录感兴趣的版本 hash
4. 再依次切换测试

### 技巧 3: 查看版本差异

```bash
# 比较两个版本的差异
git diff <commit-hash-1> <commit-hash-2>

# 查看某个版本的详细变更
git show <commit-hash>
```

### 技巧 4: 创建自定义版本标签

```bash
# 给重要版本打标签
git tag v1.0-stable

# 之后可以通过标签快速切换
git checkout v1.0-stable
```

## 最佳实践

### ✅ 推荐做法

1. **定期提交** - 每次重要更改后 commit
2. **清晰的 commit message** - 说明做了什么更改
3. **测试后再部署** - 切换到新版本后先测试
4. **保留稳定版本** - 不要删除已知稳定的版本

### ❌ 避免做法

1. **频繁切换** - 避免在短时间内多次切换版本
2. **忽略警告** - 出现错误提示时不要强行继续
3. **不保存就切换** - 切换前务必保存重要数据
4. **强制中断** - 切换过程中不要关闭程序

## 技术细节

### 实现原理

混合式版本管理器基于以下技术：

- **PyQt6 QMenu** - 下拉菜单
- **PyQt6 QDialog** - 弹窗对话框
- **subprocess** - 执行 Git 命令
- **shutil** - 文件备份和恢复

### Git 命令参考

```bash
# 获取最近 5 个版本
git log -n 5 --pretty=format:"%h|%s|%ai"

# 获取当前版本
git log -1 --pretty=format:"%h|%s|%ai"

# 切换到指定版本
git checkout <commit-hash>

# 查看详细 commit 信息
git show -s --pretty=format:%B <commit-hash>

# 查看完整日志
git log --oneline -50
```

### 数据保护机制

版本切换时的数据保护流程：

```
原始数据 → 备份到 .git_backup/ → Git checkout → 从备份恢复
```

这个机制确保：
- ✅ 用户数据不会丢失
- ✅ 配置保持不变
- ✅ 模型文件不会重复下载

## 常见问题解答 (FAQ)

### Q: 切换版本会删除我的模型吗？
**A:** 不会。系统会在切换前备份 `models/` 目录，切换后再恢复。

### Q: 可以在不同分支之间切换吗？
**A:** 可以。只要知道目标 commit 的 hash，就可以切换到任何分支的任何版本。

### Q: 切换失败后如何恢复？
**A:** 切换到其他版本即可。用户数据在 `.git_backup/` 中有备份。

### Q: 为什么下拉菜单只显示 5 个版本？
**A:** 为了保持界面简洁和快速响应。查看更多版本请点击 "查看更多版本..."。

### Q: 可以自定义显示的版本数量吗？
**A:** 可以修改 `_get_recent_versions(limit=5)` 中的 `limit` 参数。

### Q: 切换版本后需要重新下载模型吗？
**A:** 不需要。模型文件会被备份和恢复，除非目标版本明确要求不同版本的模型。
