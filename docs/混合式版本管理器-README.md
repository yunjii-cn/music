## ✨ 核心特性

### 1️⃣ 双模式设计

| 模式 | 如何触发 | 显示内容 | 适用场景 |
|------|----------|----------|----------|
| **快速模式** | 点击按钮即显示 | 最近 5 个版本 | 切换到最近的版本 |
| **完整模式** | 点击"查看更多..." | 最近 20 个版本 | 浏览历史、查找特定版本 |

### 2️⃣ 快速切换

- ⚡ **速度快** - 下拉菜单响应时间 < 100ms
- 🎯 **步骤少** - 1-2 次点击完成切换
- 💡 **信息足** - 鼠标悬停显示完整 commit 信息

### 3️⃣ 数据安全

- 🔒 **自动备份** - 切换前备份用户数据
- 🔄 **自动恢复** - 切换后恢复用户数据
- 🧹 **自动清理** - 完成后清理备份目录

**保护的数据包括：**
- `config/` - 配置文件
- `models/` - 下载的模型
- `output/` - 生成的音频
- `.env` - 环境变量

## ❓ 常见问题

### Q: 点击按钮没反应？
**A:** 检查 Git 是否已安装
```bash
where git
```

### Q: 版本列表为空？
**A:** 确认项目是 Git 仓库
```bash
git status
```

### Q: 切换失败怎么办？
**A:** 提交或暂存未提交的更改
```bash
git add .
git commit -m "保存当前工作"
```

### Q: 切换后需要重新下载模型吗？
**A:** 不需要。模型文件会自动备份和恢复。

### Q: 可以在不同分支之间切换吗？
**A:** 可以。只要知道 commit hash，就可以切换到任何版本。

```css
/* 下拉菜单 */
背景色：#2D2D2D     /* 深灰色 */
```

### 交互流程

```
用户点击按钮
    ↓
显示下拉菜单 (< 100ms)
    ↓
用户选择版本
    ↓
显示确认对话框（含版本详情）
    ↓
用户确认
    ↓
显示"正在切换..."提示
    ↓
执行：备份 → Git checkout → 恢复
    ↓
显示成功消息
    ↓
程序退出（需手动重启）
```

- **GUI 框架：** PyQt6
- **版本控制：** Git
- **编程语言：** Python 3.12+

```python

MainWindow._show_version_manager()        # 主入口
```
