# 云集智能音乐创意台 - 标准化开发工作流指南

> 本文档用于指导 AI 代理和开发者按照标准化流程进行 EXE 版本的开发和构建

##  一、项目目录结构

### 1.1 整体结构

build/                              # 【开发构建目录】所有开发活动在此进行
  README.md                       # 本说明文档
  云集智能音乐创意台-vYYYY.MM.DD.HHMM/  # 【版本化开发文件夹】
    src/                        # 源代码（从根目录复制）
    dist/                       # 构建输出（EXE 文件）
    build/                      # PyInstaller 临时构建文件
    config/                     # 配置文件
      launcher.spec           # PyInstaller 配置文件
    scripts/                    # 构建脚本
      build.py                # 自动化构建脚本
    docs/                       # 版本文档和发布说明
BAK/                                # 【备份目录】所有废弃/旧文件移至此
  old_code/                       # 旧代码备份
  old_configs/                    # 旧配置备份
  deprecated/                     # 废弃文件
launcher/                           # 【源代码】启动器源代码
  main.py                         # 主程序
  version_manager.py              # 版本管理器
acestep/                            # 【核心库】ACE-Step 核心代码
dist/                               # 【输出目录】最终 EXE 输出位置

### 1.2 核心原则

1. **所有开发活动在 build/ 目录进行** - 不要直接在根目录修改
2. **版本隔离** - 每个版本有独立的开发文件夹
3. **不删除文件** - 所有废弃文件移动到 BAK/ 目录
4. **自动版本控制** - 每次构建自动生成时间戳版本号

##  二、开发新版本完整流程

### 2.1 步骤 1：创建新版本文件夹

powershell
# 生成版本号（格式：vYYYY.MM.DD.HHMM）
version = (Get-Date).ToString('vyyyy.MM.dd.HHMM')
versionPath = .\build\云集智能音乐创意台-version

# 创建标准目录结构
New-Item -ItemType Directory -Force -Path 
    versionPath\src, 
    versionPath\dist, 
    versionPath\build, 
    versionPath\config, 
    versionPath\scripts, 
    versionPath\docs: 新功能
- fix: 修复 bug
- docs: 文档更新
- style: 代码格式（不影响代码运行）
- refactor: 重构（即不是新增功能，也不是修改 bug）
- test: 测试相关
- chore: 构建过程或辅助工具变动

示例：
build: 构建新版本 EXE v2026.03.26.1429
feat: 添加版本管理器功能
fix: 修复导入错误问题
docs: 更新开发工作流文档
refactor: 重构启动器代码结构

##  六、常见问题和解决方案

### 6.1 构建失败

问题：PyInstaller 构建失败

解决方案：
1. 检查 Python 环境
2. 清理构建缓存
3. 重新安装 PyInstaller
4. 使用调试模式重新构建

### 6.2 导入错误

问题：ImportError: attempted relative import with no known parent package

解决方案：
1. 检查 launcher.spec 中的 pathex 配置
2. 确保包含所有必要的模块路径
3. 使用绝对导入而非相对导入

### 6.3 文件丢失

问题：运行时提示文件丢失

解决方案：
1. 检查 launcher.spec 中的 datas 配置
2. 确保所有依赖文件都已添加到 added_files 列表
3. 验证文件路径是否正确
