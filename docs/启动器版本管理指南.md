## 🎯 版本命名规则

### 文件名格式

```
云集智能音乐创意台-YYYY-MM-DD_HHMM.exe

示例：
云集智能音乐创意台 -2026-03-17_1444.exe
云集智能音乐创意台 -2026-03-18_0930.exe
```

### 命名说明

| 部分 | 说明 | 示例 |
|------|------|------|
| `云集智能音乐创意台` | 固定前缀（品牌名） | - |
| `YYYY-MM-DD` | 构建日期 | 2026-03-17 |
| `HHMM` | 构建时间（时分） | 1444 = 14:44 |

**优点：**
- ✅ 一眼看出版本新旧
- ✅ 按字母排序自动按时间排列
- ✅ 无需查看文件属性知道构建时间
- ✅ 支持多版本共存（测试不同版本）

1. **打开项目根目录**

## 📦 版本管理策略

### 根目录保留策略

- ✅ **默认保留**：最近 3 个版本
- ✅ **自动归档**：旧版本移动到 `BAK/executables/root-launchers/`
- ✅ **手动清理**：运行清理脚本


## 🎯 最佳实践

### ✅ 推荐做法

1. **每次构建都创建版本化启动器**
   ```bash
   # 构建后立即运行
   python scripts\create-versioned-launcher.py
   ```

2. **定期清理旧版本**
   ```bash
   # 每周或每月运行一次
   python scripts\cleanup-old-launchers.py
   ```

3. **保留最近 3-5 个版本**
   - 便于回退测试
   - 不会占用太多空间

4. **记录版本变更**
   - 在 `docs/CHANGELOG.md` 中记录
   - 标注推荐版本

### ❌ 避免做法

1. **不要手动重命名为简单名称**
   - ❌ 云集智能音乐创意台.exe（无法区分版本）
   - ✅ 云集智能音乐创意台 -2026-03-17_1444.exe（清晰明确）

2. **不要删除所有旧版本**
   - 至少保留 3 个版本用于对比和回退

3. **不要混合不同命名规则**
   - 统一使用日期时间命名
   - 避免使用 `v1`, `v2`, `final` 等模糊命名

## 🔧 故障排查

### Q1: 找不到某个版本怎么办？

**A:** 检查 `BAK/executables/root-launchers/` 目录，旧版本会被归档到那里。

### Q2: 想保留特定版本不被清理？

**A:** 将该版本移动到其他目录，或重命名为其他名称（如加上 `keep-` 前缀）。

```bash
# 例如：保留某个重要版本
ren 云集智能音乐创意台 -2026-03-17_1444.exe keep-云集智能音乐创意台 -2026-03-17_1444.exe
```

### Q3: 如何快速找到最新版本？

**A:** 在文件资源管理器中按"修改日期"降序排序，第一个就是最新版本。

### Q4: 版本号中的时间和实际构建时间不一致？

**A:** 版本号来自原始文件名的时间戳（如 `202603171444`），如果手动复制可能不准确。建议始终使用脚本自动创建。
