## 🧪 测试步骤

### 步骤 1：检查前提条件

```bash
# 1. 确认已安装 Git
git --version
# 应该输出：git version 2.x.x

# 2. 确认项目是 Git 仓库
git status
# 应该显示当前分支和文件状态，而不是报错

# 3. 确认文件存在
ls launcher/version_manager.py
ls launcher/main.py
```

**✅ 期望结果：**
- Git 已安装
- 项目在 Git 仓库中
- 所有文件都存在

---

### 步骤 2：运行启动器测试

```bash
# 方式 1：直接运行源码
cd e:\AI应用\云集智能音乐创意台
python launcher\main.py

# 方式 2：如果已有打包的 EXE
双击 云集智能音乐创意台.exe
```

**✅ 期望结果：**
- 启动器正常打开
- 窗口标题显示"🎵 云集智能音乐创意台"
- 右上角有"📦 版本管理"按钮（蓝色）

### 步骤 3：测试版本管理按钮

**操作：**
1. 启动器打开后
2. 点击右上角的 **"📦 版本管理"** 按钮

**✅ 期望结果：**
- 弹出版本管理器窗口
- 窗口标题："📦 版本管理器"
- 顶部显示当前版本信息（绿色框）
- 中间显示可用版本列表

**❌ 如果出错：**
```
错误 1：点击没反应
→ 检查控制台是否有报错
→ 可能缺少 PyQt6 依赖

错误 2：提示"无法获取当前版本"
→ 确认项目在 Git 仓库中
→ 检查 .git 目录是否存在

错误 3：版本列表为空
→ 运行 git log 查看是否有提交历史
→ 至少需要有 1 个 commit
```

## 🔍 常见问题排查

### 问题 1：按钮不显示或点击没反应

**诊断：**
```bash
# 检查导入是否正常
python -c "from launcher.version_manager import VersionManagerDialog; print('OK')"
```

**解决：**
```bash
# 如果报错，检查文件路径
ls launcher/version_manager.py

# 或者重新创建文件
# （复制之前的代码）
```

---

### 问题 2：提示"不是 Git 仓库"

**诊断：**
```bash
git status
```

**解决：**
```bash
# 如果是压缩包用户，需要重新克隆
git clone <仓库地址>

# 或者初始化新的仓库
git init
git add .
git commit -m "初始提交"
```

---

### 问题 3：版本列表显示乱码

**原因：** commit message 包含特殊字符或中文编码问题

**解决：**
```bash
# 设置 Git 编码
git config --global core.utf8 true

# 或在代码中处理编码
# 修改 launcher/version_manager.py
# 在 subprocess.run 中添加 encoding='utf-8'
```

---

### 问题 4：切换版本后启动器崩溃

**原因：** 版本不兼容或依赖缺失

**解决：**
```bash
# 1. 切换到其他版本
git checkout main  # 或其他稳定分支

# 2. 重新安装依赖
pip install -r requirements.txt

# 3. 重新打包
cd launcher
python build.py
```

## 🎯 性能测试（可选）

### 加载速度测试

**目标：** 版本管理器打开时间 < 2 秒

**方法：**
```bash
# 使用计时器
import time
start = time.time()
dialog = VersionManagerDialog()
end = time.time()
print(f"加载时间：{end - start:.2f}秒")
```

**标准：**
- ✅ < 1 秒：优秀
- ✅ 1-2 秒：良好
- ⚠️ 2-5 秒：可接受
- ❌ > 5 秒：需要优化

---

### 内存占用测试

**目标：** 版本管理器内存占用 < 50MB

**方法：**
```bash
# 使用任务管理器查看
# 或使用 Python 工具
import psutil
process = psutil.Process()
print(f"内存占用：{process.memory_info().rss / 1024 / 1024:.2f} MB")
```
