## 🔧 功能说明

### 1. 查看当前版本

- **位置**：窗口顶部绿色区域
- **显示内容**：
  - ✅ 当前使用的版本号（Git commit hash）
  - 📝 版本说明（commit message）
  - 📅 构建日期时间

### 2. 查看可用版本

- **位置**：窗口中部列表
- **显示内容**（每个版本）：
  - 📋 版本号
  - 📝 版本说明
  - 📅 构建日期
  - [👁️ 预览] 按钮 - 查看详细变更
  - [🔄 切换] 按钮 - 切换到该版本

### 3. 预览版本详情

**操作步骤：**
1. 找到想查看的版本
2. 点击 [👁️ 预览] 按钮
3. 弹出详细信息窗口

**显示内容：**
```
┌─────────────────────────────────────┐
│ 📋 7d8c91a                          │
│ 📅 2026-03-19 15:20                 │
├─────────────────────────────────────┤
│ 变更说明：                          │
│                                     │
│ 添加批量生成功能，支持同时生成 5 首音乐 │
│ 优化内存占用，减少 20%               │
│ 修复已知 bug                        │
└─────────────────────────────────────┘
```

### 4. 切换版本

**操作步骤：**
1. 找到想切换的版本
2. 点击 [🔄 切换到此版本] 按钮
3. 确认对话框 → 点击 [是]
4. 等待切换完成提示
5. **重启启动器**

**注意事项：**
- ⚠️ 切换前会自动备份用户数据（配置、模型等）
- ⚠️ 切换后必须重启启动器才能生效
- ⚠️ 如有未保存的工作，请先保存

### 5. 刷新版本列表

**何时需要刷新：**
- 刚拉取了新的代码
- 切换版本后想看到最新状态

**操作：** 点击 [🔄 刷新] 按钮

### 6. 查看完整日志

## 🎯 常见使用场景

### 场景 1：回退到稳定版本

**问题：** 更新后发现新版本有 bug

**解决步骤：**
```
1. 打开版本管理器
2. 找到之前的稳定版本（看日期和说明）
3. 点击 [🔄 切换到此版本]
4. 确认切换
5. 重启启动器
✅ 完成！已回到稳定版本
```

### 场景 2：对比两个版本

**问题：** 想知道新旧版本有什么区别

**解决步骤：**
```
1. 打开版本管理器
2. 查看当前版本说明（顶部）
3. 找到想对比的旧版本
4. 点击 [👁️ 预览] 查看详细说明
5. 根据说明决定是否切换
```

### 场景 3：测试最新开发版

**问题：** 想尝试最新的功能

**解决步骤：**
```
1. 确保已拉取最新代码：git pull
2. 打开版本管理器
3. 点击 [🔄 刷新]
4. 第一个版本就是最新的
5. 点击 [🔄 切换到此版本]
6. 重启启动器测试
```

### 场景 4：查看某个功能的引入版本

**问题：** 想知道某个功能是什么时候添加的

**解决步骤：**
```
1. 打开版本管理器
2. 点击 [📝 查看完整日志]
3. 浏览历史记录，查找相关说明
4. 找到后可以切换到那个版本查看
```

## ⚠️ 注意事项

### 1. 数据保护

**切换版本时会自动：**
- ✅ 备份 `config/` 目录（配置文件）
- ✅ 备份 `models/` 目录（模型文件）
- ✅ 备份 `output/` 目录（生成的音频）
- ✅ 备份 `.env` 文件（环境变量）
- ✅ 切换后自动恢复这些数据

**不会备份的：**
- ❌ 临时文件
- ❌ 日志文件
- ❌ 缓存文件

### 2. Git 要求

**系统必须安装：**
- ✅ Git（版本 >= 2.0）
- ✅ 项目必须是 Git 仓库

**检查方法：**
```bash
# 检查 Git 是否安装
git --version

# 检查是否在 Git 仓库中
git status
```

**如果报错 "不是 Git 仓库"：**
```
可能原因：
1. 你是从压缩包解压的，不是 git clone 的
2. .git 目录被删除了

解决方法：
git clone <仓库地址>
```

### 3. 权限要求

**切换版本需要：**
- ✅ 对项目的写入权限
- ✅ 能够执行 git checkout 命令

**如果有未提交的更改：**
```
⚠️ 警告：你有未提交的更改

选项：
- 先提交或暂存更改
- 或者放弃更改继续切换
```

## 🐛 故障排查

### 问题 1：点击版本管理按钮没反应

**可能原因：**
- 版本管理器模块未正确导入

**解决方法：**
```bash
# 检查文件是否存在
ls launcher/version_manager.py

# 重新运行启动器
python launcher/main.py
```

### 问题 2：提示"无法获取当前版本"

**可能原因：**
- Git 未安装
- 不在 Git 仓库中
- .git 目录损坏

**解决方法：**
```bash
# 1. 检查 Git 是否安装
git --version

# 2. 检查是否在仓库中
git status

# 3. 如果是压缩包用户，需要重新克隆
git clone <仓库地址>
```

### 问题 3：切换版本失败

**可能原因：**
- 有未提交的更改
- 文件被占用
- 权限不足

**解决方法：**
```bash
# 1. 查看当前状态
git status

# 2. 如果有未提交更改，选择：
#    - 提交更改：git add . && git commit -m "message"
#    - 放弃更改：git checkout .

# 3. 重试切换
```

### 问题 4：切换后启动器无法启动

**可能原因：**
- 版本不兼容
- 依赖缺失

**解决方法：**
```bash
# 1. 切换到其他版本
python scripts/switch-version.py <其他版本 hash>

# 2. 或手动恢复
git checkout <好的版本>

# 3. 重新安装依赖
pip install -r requirements.txt
```

## 💡 高级技巧

### 技巧 1：快速返回上一个版本

```bash
# 命令行方式
git checkout HEAD~1

# 或在版本管理器中：
# 1. 找到第二个版本
# 2. 点击切换
```

### 技巧 2：同时打开多个版本对比

```bash
# 方式 1：使用不同端口
# 修改 .env 中的端口配置
# 然后启动两个实例

# 方式 2：复制项目
cp -r project project-copy
# 在副本中切换到不同版本
# 分别启动对比
```

### 技巧 3：保留常用版本

```bash
# 给版本打标签
git tag -a stable-v1 7d8c91a -m "稳定版 v1"

# 以后可以快速切换
git checkout stable-v1
```
