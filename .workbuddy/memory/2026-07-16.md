## 修复：模型管理页『不完整』模型不显示『重新下载』按钮（fix/model）
- 现象：用户追问为何『不完整』(integrity_status=="incomplete") 模型看不到『重新下载』。
- 根因（与上一处同源）：`_load_model_list` 中，子模型的
  `exists = check_model_exists(name)` 返回 **False**（该函数要求文件齐全 + 大小达标），
  而 `integrity_status` 在目录存在但不完整时返回 **"incomplete"**。于是 incomplete 模型 =
  `exists=False && integrity_status="incomplete"`。但按钮分支把 incomplete 写在
  `elif model["exists"]` 之下 → 落入空分支、无任何按钮（连『下载』都没有）。
  （main 模型在 _load_model_list 中 integrity_status 只有 complete/missing 两态，不会是 incomplete。）
- 修复（commit af06f18）：
  - version_manager.py（真正渲染 UI）：将 incomplete 分支独立为 `elif integrity_status == "incomplete"`，
    不再依赖 model["exists"]。
  - main.py（遗留方法）：外层 guard 改为 `elif model["exists"] or integrity_status == "incomplete"`
    （最小改动、不动缩进）。
  - 行为：未安装→下载；不完整→删除+重新下载；完整→删除（主组件额外重下）；下载中→暂停。py_compile 通过。
- 仅本地 commit，未 push。仍提醒用户：需重新打包 exe 或本地 python 运行才能验证。
