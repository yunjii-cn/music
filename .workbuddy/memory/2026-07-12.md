## 模型管理器 bug 修复：损坏/不完整模型既无法删除也无法重新下载

**现象**：模型管理器里，不完整或（通过文件名+大小校验误判为“已安装”的）损坏模型，主模型组件（acestep-v15-turbo、acestep-5Hz-lm-1.7B）的「删除」按钮被禁用，且只显示「删除」不显示「下载」，陷入死局。

**根因**：`version_manager.py` 的 `ModelManagerDialog._update_ui` 与 `main.py` 的 `_update_model_management_ui` 仅按 `model["exists"]`（二值）渲染按钮（exists→删除，否则→下载），没有“不完整”这一中间态的分支。主组件删除被 `setEnabled(False)`，且主组件不在 `SUBMODEL_REGISTRY` 中，点「下载」走 CLI 会报 “Unknown model”。下载链路也从不传 `--force`，即便能重下也无法覆盖损坏文件。

**修复**（dev/app/ 下三处）：
1. `acestep/model_downloader.py`：`download_main_model` / `download_submodel` 在 `force=True` 时，先删除不完整/损坏的（主组件/子模型）目录再下载，保证全新重装；主模型只清掉无效组件、保留完好组件。
2. `main.py`：`ModelDownloadThread` 与 `_download_model` 新增 `force` 参数，force 时给 CLI 加 `--force`。
3. `version_manager.py` 与 `main.py` 两处 UI：按 `integrity_status` 三态分支渲染——“complete”→删除（主组件仍禁用，但额外给主组件显示「重新下载」）；“incomplete”→同时给「删除」+「重新下载(force)」；“missing”→「下载」。主组件的重下统一路由到 `download_main_model`（即模型名传 "main"）。

**验证**：py_compile 三文件通过；离线测试确认 `check_model_exists` 对部分目录返回 False、`delete_model` 能删部分目录、force 移除逻辑正确。GUI 需在用户本机实际跑一次确认。

## 启动器顶栏：新增「收起到托盘」按钮 + 关闭按钮改为完整退出

**需求**：顶栏（app 可见的顶部导航栏 `nav_bar`，非 OS 原生标题栏）增加「最小化到托盘」按钮；点窗口关闭(X)改为“关闭启动器并后台终止所有项目进程”。

**现状**：`MainWindow` 用原生 OS 标题栏（无自定义窗口按钮）；系统托盘 `QSystemTrayIcon` 已在 `_setup_tray()`（main.py:2661）就绪。原 `closeEvent`（main.py ~7598）默认 `hide()` 到托盘、进程继续；真正清理全部进程仅在托盘「退出」→ `_exit_all()`（main.py:7561，含 `_stop_all_services` + psutil 扫描终止 python/node/powershell 中含 acestep/qinglong 的进程）。

**修改**（dev/app/main.py）：
1. `__init__` 新增 `self._exiting = False` 守卫。
2. 在 `nav_bar` 右侧（4 个页面按钮之后 `addStretch(1)`）新增「🗕 收起到托盘」按钮 `btn_min_tray`，tooltip 说明后台继续运行，点击 `_minimize_to_tray()`（隐藏窗口+托盘提示，进程不杀）。
3. 新增 `_minimize_to_tray()` 方法（原 closeEvent 的 hide 到托盘逻辑抽出）。
4. `_exit_all()` 增加 `self._exiting` 守卫防重入，其余逻辑不变。
5. `closeEvent` 改写：非 `_force_exit` 时（即用户点 X）直接调用 `_exit_all()` 执行完整退出（终止所有项目进程后关闭）；`_force_exit` 分支维持最终清理（monitor 停止、tray 隐藏、accept）。

**行为对照**：X 按钮 = 完整退出（杀光项目进程）；托盘按钮 = 仅收起（进程后台运行）；托盘右键「退出」= 完整退出（不变）。
**验证**：py_compile main.py 通过；全仓仅 `_exit_all` 内一处 `self.close()`，无其它调用者受新 close 行为影响。GUI 行为需本机实跑确认。
