# 2026-07-02 工作日志

## 底部按钮智能状态切换 + Qt CSS 清理

### 修复 bat 无法启动问题（2:00）
- **根因**：按钮样式中的 `transition` / `transform` / `box-shadow` 是 Web CSS3 属性，Qt StyleSheet 不识别，触发了大量 `Unknown property` 警告。虽然不致命，但可能被误判为错误
- **修复**：用 Python 脚本批量从所有按钮样式中移除了这三类无效声明
- **验证**：移除后 app.exec() 正常挂起并返回 0（在 offscreen 模式下测试通过）

### 底部按钮合并为智能单按钮（2:30）
- **原设计**：三栏等分（重启 | 启动 | 停止），始终显示
- **新设计**：单一主按钮 + 条件显示管理按钮

| 状态 | 主按钮（居中） | 副按钮 |
|------|:---:|:---:|
| 空闲（无服务运行） | 🎵 **启动音乐创意台**（红色，48px高） | 🔄重启 / ⏹停止 → 隐藏 |
| 启动中 | ⏳ **正在启动...**（橙色，禁用） | 隐藏 |
| 运行中（有服务） | 🎵 **音乐创意台运行中**（绿色，禁用） | 🔄重启（左）/ ⏹停止（右）→ 显示 |

- `_on_status_changed` 中维护 `_running_services_count` 计数器
- `_update_bottom_bar()` 统一管理按钮显示/隐藏/文字/颜色
- 移除了所有对旧按钮名 `btn_deploy_maintain` / `btn_stop_all` 的引用，替换为 `btn_bottom_restart` / `btn_bottom_stop`
- `_enable_buttons()` 和 `_start_project()` 现在通过 `_update_bottom_bar()` 统一处理状态切换
