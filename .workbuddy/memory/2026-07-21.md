## 关键纠正：用户"和原目录一致" = 开发指南的用户运行环境目录结构（非 .py 源文件）

## self-deploy 两轮修复：206 无限嵌套 → 正式改用「固定名入口=版本号exe 启动器」ランチャーパターン（20:00）
- **用户反馈①**：双击 exe 后"反复弹正在初始化启动屏近 20 次"，最后报 `FileNotFoundError: [WinError 206] 文件名或扩展名太长`（launcher.py _self_deploy 里 subprocess Popen 处）。
  - **根因**：旧 `_self_deploy` 把 `deploy_dir = dirname(exe) + BRAND_NAME`。固定名入口从 `品牌/品牌.exe` 运行时，deploy_dir 被算成 `品牌/品牌/`（自身子文件夹）→ 判定"未部署"→ 再往更深一层部署 → 每层新起一个 onefile 进程(新启动屏) → 路径越来越长直到 >260 字符 206。
  - **首轮修复**：新增 `_resolve_deploy_dir()` 向上走查找已存在的「品牌名」目录（含 ver/ 子级），避免把部署目录算成自身子文件夹。
- **用户反馈②**："**固定名入口打不开版本号exe**"——说明设计本身不对。
  - **关键发现（读参考项目源码）**：云集智能文件清理专家的 **launcher.py 根本不做 self-deploy**！self-deploy 在 `downloader.py::_self_deploy`，由 `_find_dev_dir()` 仅在"找不到 LOCK/app"时才调用（固定名入口在自己目录找到 LOCK 即直接 return，绝不递归）。真架构是**「固定名入口=启动器」**：`_find_current_version_exe(ver_dir)` 找 ver/ 里最新版本号 exe → `_launch_app()`=`Popen([ver_exe], cwd=dev_dir)` 启动它；版本号 exe 才是真正的 app。
  - 本项目 `main.py` 早有此预期：`get_install_root()`/`get_version_from_filename()` 注释写"固定名入口回退读安装根目录 version.txt"，`version_manager` 需 ver/ 多版本切换 → 印证「入口启动 ver/版本号exe」才对。旧实现把入口做成版本号exe硬链接直接跑 main = 错，正是"固定名入口打不开版本号exe"病根。
- **launcher.py 全面改写为「ランチャーパターン」`_dispatch()` 三分支**（`frozen` 时执行，随后 `import main; main.main()`）：
  - **A) 版本号 exe 在 部署目录/ver/ 内**（`is_versioned and exe_in_ver and in_deploy`）→ set `YUNJI_INSTALL_ROOT` → return 落到 `main.main()` 跑真正 app（无递归）。
  - **B) 固定名入口**（`exe_name==品牌名.exe and in_deploy`）→ `_find_current_version_exe(ver_dir)` → `_launch_versioned()`=`Popen([ver_exe]+透传参数, cwd=deploy_dir, CREATE_NO_WINDOW)` 启动它 → `os._exit(0)`；ver/ 无可用 exe 时入口自身兜底跑 main。
  - **C) 便携版本号 exe（未部署，如 Downloads）**→ `_self_deploy()`：建 `品牌/ver+app+data+temp` + `.yunji.lock` + `version.txt` + `ver/current.txt`，复制自身进 `ver/品牌-v<ver>.exe`，建固定名入口，`Popen([entry,"--cleanup="+原exe])` 拉起入口 → `os._exit(0)`。已部署(D)时直接把外部 exe 并入 ver/ 并委托入口，**不再嵌套**。
  - 辅助：`_find_current_version_exe` 优先读 `ver/current.txt`，否则按正则 `v?(\d{4}\.\d{2}\.\d{2}\.\d{4})` 取最新；`--cleanup` 收敛为 `_get_cleanup_path()/_do_cleanup()`；`_passthrough_args()` 转发参数(剔 --cleanup)。
- **验证**：干跑 `_test_launcher.py` 覆盖 A/B/C/D 四分支 **25/25 断言全绿**（B 入口确实启动 ver/版本号exe、A 直接跑 main 无子进程、D 无嵌套）。已删测试文件。
- **重建**：`build_release.py` 后台重打包（task jt3lYV），产 `v2026.07.21.<新时刻>.exe`。
- ⚠️ 沙箱无法真正双击 GUI exe，需用户本机实测：双击版本号 exe → 建文件夹 → 固定名入口启动 ver/版本号exe → app 出现、原便携 exe 被删。
