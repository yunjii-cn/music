# 项目整合计划

## 目标
将官方演练场（Gradio UI）和青龙训练器（React UI）整合成一个项目的两种界面，共享资源和配置，避免重复文件和目录。

## 已完成的整合

### 1. 共享模型存储
- 修改了 `acestep/model_downloader.py`：使用根目录的 `checkpoints` 文件夹
- 修改了 `acestep/core/generation/handler/init_service_catalog.py`：使用根目录的 `checkpoints` 文件夹
- 两个项目现在共享同一个模型存储目录

### 2. 共享输出文件夹
- 修改了 `acestep/api_server.py`：使用根目录的 `output` 文件夹
- 两个项目现在共享同一个输出目录

### 3. 统一项目根目录
- 修改了 `acestep/core/generation/handler/progress.py`：使用统一的根目录计算方式

## 待完成的整合

### 4. 共享配置文件
- 创建根目录的 `config` 文件夹，存放共享配置
- 实现统一的配置管理系统
- 支持环境变量和配置文件的优先级管理

### 5. 统一启动脚本
- 创建 `start-all.ps1` 和 `start-all.sh` 脚本
- 同时启动 API 服务器和前端开发服务器
- 提供统一的停止脚本

### 6. 共享工具函数
- 创建 `shared` 目录，存放共享的工具函数
- 提取两个项目中重复的代码
- 实现统一的日志、路径管理等功能

### 7. 统一设置管理
- 创建统一的设置存储系统
- 支持设置的持久化和加载
- 提供设置同步功能

### 8. 共享数据目录
- 统一数据集存储目录
- 统一缓存目录
- 统一临时文件目录

## 项目结构优化

### 推荐的目录结构
```
e:\AI应用\qinglong-music-trainer-2.8.3/
├── acestep/          # 后端核心代码
├── ace-step-ui/      # 青龙训练器前端
├── checkpoints/      # 共享模型存储
├── config/           # 共享配置
├── data/             # 共享数据
│   ├── datasets/     # 数据集
│   ├── cache/        # 缓存
│   └── temp/         # 临时文件
├── output/           # 共享输出
├── shared/           # 共享工具
├── scripts/          # 脚本
│   ├── start-all.ps1 # 启动所有服务
│   ├── start-all.sh  # 启动所有服务
│   └── stop-all.sh   # 停止所有服务
└── README.md         # 项目说明
```

## 实施步骤

1. **创建共享目录结构**
   - 创建 `config`、`data`、`shared` 目录
   - 配置目录权限

2. **实现配置管理**
   - 创建配置加载和保存模块
   - 支持不同环境的配置

3. **创建统一启动脚本**
   - 编写 `start-all` 脚本
   - 实现服务的并行启动

4. **提取共享工具**
   - 识别重复代码
   - 创建共享工具模块

5. **测试整合**
   - 测试模型共享
   - 测试输出共享
   - 测试配置同步

## 预期效果

- **资源共享**：模型、配置、数据等资源在两个项目间共享
- **统一管理**：通过统一的脚本和配置进行管理
- **减少重复**：消除代码和资源的重复
- **简化部署**：统一的启动和停止流程
- **一致性**：两个界面使用相同的后端和资源

## 技术优势

- **节省存储空间**：模型只需要下载一次
- **提高开发效率**：共享代码和配置
- **简化维护**：统一的代码库和配置管理
- **一致的用户体验**：两个界面使用相同的后端服务
